# Hybrid Video Editor Pro (Python + Remotion + FFmpeg)

## 🚨 Critical Architecture Overview

This project **IS NOT** a standard Remotion project. It uses a hybrid rendering pipeline for maximum performance.

### The Pipeline

1.  **Ingestion & Processing (Python)**
    -   Script: `video-editor-pro/scripts/orchestrator.py`
    -   Logic: Removes silence (Silero VAD) and applies dynamic zoom (FFmpeg).
    -   Output: A processed `.mp4` video file (e.g., `video_zoomed.mp4`).

2.  **Configuration (JSON)**
    -   File: `editor-master/remotion_input.json`
    -   Logic: Stores transcript, subtitles, and motion graphics data generated by Python.

3.  **Visual Overlays (Remotion)**
    -   App: `editor-master/`
    -   Logic: Renders *only* transparent layers (subtitles, animations).
    -   **Important:** The base video is treated as a static background layer.

4.  **Final Rendering (FFmpeg Composition)**
    -   Script: `editor-master/fast_render.py`
    -   Logic: Composites the processed video + Remotion overlays.
    -   **Benefit:** 10x faster than full Remotion rendering.

## 🚀 Commands

### 1. Full Auto-Edit (Silence + Zoom + Subtitles)
```bash
python3 video-editor-pro/scripts/orchestrator.py --input video.mov --edit --subtitles
```

### 2. Preview (Remotion Studio)
```bash
cd editor-master && npm run dev
```

### 3. Final Render
```bash
python3 editor-master/fast_render.py <PROCESSED_VIDEO_NAME>
```
